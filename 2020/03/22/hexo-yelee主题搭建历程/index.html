发布到 Github Pages

参考了 这里

这里遇到了两个问题

  1. PowerShell 无法正常运行

    具体的报错信息无法考证了，可以这样解决

    • 在设置里搜索 PowerShell ，应用允许本地 PowerShell 脚本在不签名的情况下运行
    • 改用 GitBash 执行命令

  2. 使用 hexo d 命令部署到 github pages 时提示

    ERROR: Repository not found.

    fatal: Could not read from remote repository.

    可能原因是

    • 配置 SSH key 时出错，重置 SSH key 可以使用 ssh -T git@github.com 检验是否配置成功

    • 检查博客根目录下配置文件 _config.yml 的站点域名是否正确

    • 检查网络

      传言原因之一可能是git被墙，不过本人只是没有正确配置域名而已(((

再次配置的过程中遇到几个问题

  1. 在匹配SSH时输入 ssh -T git@github.com 后突然就鬼畜了，输出一堆 yyyyyyyyy ……

    解决办法：先输入一个 y 再输入 yes 就好了…神奇的bug

  2. 修改npm全局目录的时候没仔细看文件名，乌龙了半天…

    npm环境变量配置

  3. 使用 hexo s 测试时发现首页不显示文章，排查后发现这是作者遗留的一个bug

    解决方法：

    修改themes/yelee/layout/_partial/head.ejs中的这段代码

    把这段代码的第12行改为

    search: <%= theme.search.onload %>

    参考：https://blog.csdn.net/youshaoduo/article/details/78709160

  4. 配置完主题设置后发现初始化页面失败，排查过后应该还是作者遗留的bug

    解决方法：不要修改 theme\yelee\_config.yml 配置中的 #friends: false 和 #aboutme: false

  5. 关于主题侧边栏社交图标中的github图标不显示的问题，解决如下：

6.10更新

各位能不用这个主题就不用这个主题吧，，，作者跑路了很久了，这个主题实在没有前途，如果喜欢折腾和DIY主题的大佬当我没说。。。

各位可以考虑用这个主题的原生主题 yilia 或者经过dalao改版的 yelee 主题，或是派生主题之 yelog， 或者干脆放弃以yilia为原型的双栏主题，使用更酷炫的 NexT 等等高人气主题
